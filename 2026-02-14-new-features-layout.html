Architecture Notes

Feature 3 ‚Äì AI SQL Tool: The chat backend uses an LLM with a tool/function that can run read-only SQL queries against the prompt_responses table (and any future tables). The LLM decides when to query, formulates SQL, and interprets results in natural language.
Feature 4 ‚Äì Auth: Username/password stored server-side (hashed). Token stored in iOS Keychain, shared with Watch via WatchConnectivity or shared Keychain group. All API calls get an Authorization header.
Feature 6 ‚Äì Prompt Flow Config: Flow definition stored as JSON on the server per user. Each step has: prompt text, answer buttons (label + destination step or "submit"), and a step type (prompt, reflection, voice_note). iOS and Watch fetch config on launch, cache locally.
Feature 7 ‚Äì Timers: Pure client-side on Watch. Uses WKExtendedRuntimeSession or background refresh for haptic vibration when timer ends. No server component.

UX Behavior Notes

Keyboard Behavior
When the chat input is tapped, the keyboard slides up and the message list scrolls to keep the latest message visible. The input bar stays pinned directly above the keyboard. Preset question chips are hidden when the keyboard is open, replaced by the scrolled message history. Dismissing the keyboard (tap outside or swipe down) restores the full layout.
Watch Notification Sync
Notification schedules are generated on the iPhone and synced to the Watch via WatchConnectivity (transferUserInfo or applicationContext). When the user changes notification settings on iPhone, the new schedule is pushed to the paired Watch. The Watch uses this schedule to fire local notifications independently, even when the iPhone is out of range. On Watch app launch, it also requests the latest schedule from the iPhone if available.
Step Deletion Behavior
When a step is deleted, remaining steps are renumbered sequentially (Step 3 becomes Step 2, etc.). Any answers in other steps that pointed to the deleted step become dangling references ‚Äî these are highlighted with a warning badge in the editor and default to "Submit" until the user re-assigns them. A confirmation dialog warns: "Step 3 is referenced by 2 answers in other steps. Delete anyway?"
